# Memory Bank - Kalvis Lejasmeiers Architect Portfolio

## Overview
This memory bank serves as the central documentation system for the architect portfolio website project. It tracks project context, decisions, progress, technical specifications, and content requirements.

## File Structure

### 1. `project_context.md`
- **Purpose:** Core project requirements and design philosophy
- **Contents:** Project overview, goals, design direction, site structure
- **Use:** Reference for understanding the "why" behind design decisions

### 2. `progress_tracker.md`
- **Purpose:** Track development progress across all phases
- **Contents:** Checklist of tasks organized by development phases
- **Use:** Monitor what's completed, in progress, and pending

### 3. `decisions_log.md`
- **Purpose:** Document key decisions and their rationale
- **Contents:** Chronological log of design and technical decisions
- **Use:** Understand why specific approaches were chosen

### 4. `technical_specs.md`
- **Purpose:** Detailed technical implementation specifications
- **Contents:** File structure, HTML/CSS/JS architecture, performance targets
- **Use:** Development reference for implementation details

### 5. `content_inventory.md`
- **Purpose:** Track all content requirements and placeholders
- **Contents:** Navigation text, project listings, about/contact content
- **Use:** Content management and copywriting reference

## How to Use This Memory Bank

### For Project Managers
1. Review `project_context.md` for project goals and requirements
2. Check `progress_tracker.md` for current status
3. Consult `decisions_log.md` for historical context

### For Developers
1. Start with `technical_specs.md` for implementation details
2. Reference `project_context.md` for design philosophy
3. Update `progress_tracker.md` as tasks are completed
4. Document decisions in `decisions_log.md`

### For Content Creators
1. Use `content_inventory.md` for all text requirements
2. Reference `project_context.md` for tone and style guidelines
3. Check `technical_specs.md` for image specifications

## Update Workflow
1. **Daily:** Update `progress_tracker.md` with completed tasks
2. **When making decisions:** Document in `decisions_log.md`
3. **When changing specs:** Update `technical_specs.md`
4. **When content changes:** Update `content_inventory.md`
